# Спецификация MITM-скрипта

> Документ для разработчика. Заполняется по логам из `data/logs/<bank>.json`.
> Когда документ готов — кидаешь его в чат, по нему пишется proxy-скрипт и API.

---

## Как заполнять (кратко)

1. Открыть приложение банка через прокси.
2. Найти в логах нужный экран (главная, карта, история и т.д.).
3. Записать **роут**, **метод**, **хост**, **статус**.
4. Вставить **фрагмент response** и указать **какие поля менять**.
5. Описать **откуда брать новое значение** (наше API / формула / константа).
6. Перечислить **правила и ограничения** (единицы, минус, связанные поля).
7. Указать поля, которые **трогать нельзя** (похожие названия, но другой смысл).

# ПРИМЕР (заполнен): Rocket Bank — изменение баланса

## 1. Общая информация

| Поле | Значение |
|------|----------|
| Банк | Rocket Bank |
| ID модуля в коде | `rocketbank` |
| Хост приложения | `dbo.rocketbank.ru` |
| Файл логов | `data/logs/rocketbank.json` |
| Что подменяем | Баланс на главном экране и на экране карт |
| Тип изменения | `response_replace` — меняем тело ответа банка |

## 2. Откуда берётся новое значение

| Поле | Значение |
|------|----------|
| Источник | Наше API / БД |
| Эндпоинт установки | `PATCH /banks/rocketbank/config/balance` |
| Тело запроса | `{ "balance": 99999.50 }` |
| Поле в конфиге | `balance` |
| Тип | `float64` |
| Единицы | **Рубли с копейками** (не копейки!) |
| Пример | `100` = 100 ₽, `22096.74` = 22 096,74 ₽ |

## 3. Роуты для подмены

### 3.1. Общий баланс (виджет на главной)

| Поле | Значение |
|------|----------|
| Метод | `GET` |
| Роут | `/v1/products/balances` |
| Query | — |
| Статус | `200` |
| Когда срабатывает | Открытие главного экрана, pull-to-refresh |

**Фрагмент лога (`data/logs/rocketbank.json`):**

```json
{
  "method": "GET",
  "host": "dbo.rocketbank.ru",
  "route": "/v1/products/balances",
  "status": 200,
  "response": [
    {
      "balance": 100,
      "currency": "810",
      "currencyInfo": {
        "code": "RUR",
        "symbol": "₽"
      },
      "title": "Общий баланс:"
    }
  ]
}
```

**Что менять:**

| JSON-путь | Поле | Действие | Новое значение |
|-----------|------|----------|----------------|
| `[0].balance` | `balance` | заменить | значение из конфига `balance` |

**Что НЕ менять:**

- `currency`, `currencyInfo`, `title` — оставить как в оригинале

---

### 3.2. Баланс карты и счёта

| Поле | Значение |
|------|----------|
| Метод | `GET` |
| Роут | `/v1/cards/list` |
| Query | опционально `shakeIt=true` (не важно для подмены) |
| Статус | `200` |
| Когда срабатывает | Экран «Карты», обновление списка карт |

**Фрагмент лога:**

```json
{
  "method": "GET",
  "host": "dbo.rocketbank.ru",
  "route": "/v1/cards/list",
  "status": 200,
  "response": [
    {
      "accountInfo": {
        "accountName": "Мой счёт",
        "accountNumber": "40817810350224532469",
        "balance": 100,
        "cards": [
          {
            "balance": 100,
            "cardType": "DEBIT",
            "cashBackInfo": {
              "cashBackBalance": 0
            },
            "creditBalance": 0
          }
        ]
      }
    }
  ]
}
```

**Что менять:**

| JSON-путь | Поле | Действие | Новое значение |
|-----------|------|----------|----------------|
| `[].accountInfo.balance` | `balance` | заменить | значение из конфига |
| `[].accountInfo.cards[].balance` | `balance` | заменить | то же значение |

**Правило поиска полей:**

- Менять **только** ключ `"balance"` рекурсивно по всему JSON.
- Поля с похожими названиями **не трогать**:
  - `cashBackBalance`
  - `creditBalance`
  - `scoresBalance`
  - `totalScoresBalance`

## 4. Условия срабатывания скрипта

```
IF method == GET
AND host == dbo.rocketbank.ru
AND status == 200
AND route IN (/v1/products/balances, /v1/cards/list)
AND config.balance IS NOT NULL
THEN подменить поля balance
ELSE не трогать ответ
```

## 5. Правила и валидация

| # | Правило |
|---|---------|
| 1 | Баланс хранится в **рублях** (float), не в кopeйках |
| 2 | Допускаются дробные значения: `99999.50` |
| 3 | Отрицательный баланс **можно** записать в конфиг (ограничений нет) |
| 4 | Если `balance` в конфиге `null` — скрипт **не применяется**, ответ банка без изменений |
| 5 | Все поля `balance` в одном ответе получают **одно и то же** значение |
| 6 | Ответ может быть `gzip` — нужно распаковать, изменить, упаковать обратно |
| 7 | Если кодировка не `gzip` и не пустая — не трогать |

## 6. Проверка (как убедиться что работает)

1. `PATCH /banks/rocketbank/config/balance` → `{ "balance": 50000 }`
2. Открыть Rocket Bank через прокси
3. На главной виджет «Общий баланс» → **50 000 ₽**
4. На экране карт баланс счёта и карты → **50 000 ₽**
5. В логах сервера: `rocketbank balance change applied: route=/v1/products/balances balance=50000.00`

## 7. Связанные скрипты (не входят в эту задачу)

- История операций — отдельная спецификация
- ФИО / телефон / номер карты — отдельная спецификация
- PDF-чеки — отдельная спецификация
